### 记录 003（数据库闭环审查与修订）

- 背景：
  - 用户要求基于需求文档检查数据库方案草案，确认是否无误、是否闭环。
  - 审查结果：原草案方向正确，但存在关键约束缺口，尚未达到“完美闭环”。
- 修订动作（已完成）：
  - 将草案升级为 `Draft v0.2`。
  - 补齐单写入端归属约束：`users(user_id, writer_device_id)` 复合外键绑定设备归属。
  - 补齐迁移设备归属约束：`migration_requests` 的 `from/to` 设备均受 `user_id` 复合外键约束。
  - 增加 `uk_migration_user_pending`：同一用户同一时刻只允许一个 `PENDING` 迁移。
  - 增加分钟精度与本地日期一致性约束：`at_utc` 分钟截断、`local_date` 与时区换算一致、`minute_of_day` 一致。
  - 增加请假日与汇总口径约束：`is_leave_day=true` 时 `adjust_minutes=0`，并约束 `INCOMPLETE/COMPUTED` 字段完整性。
  - 增加限流承载表：`security_attempt_windows`。
  - 增加同步幂等与原子提交承载表：`sync_commits`。
  - 增加跨行业务规则触发器建议：`END > START`（约束触发器，事务延迟校验）。
- 影响：
  - 数据层对需求文档中的一致性与验收项覆盖度显著提升，后续可直接进入 API 设计与实现阶段。

### 记录 004（数据库闭环复查与二次修订）

- 背景：
  - 用户再次要求按需求文档复查“是否无误、是否完美闭环”。
  - 复查发现仍有 4 个缺口：`END` 无 `START` 未阻断、请假日 `INCOMPLETE` 与 `adjust=0` 约束冲突、日/月汇总缺少 `version`、Web 只读绑定缺少持久化模型。
- 修订动作（已完成）：
  - 将草案升级为 `Draft v0.3`。
  - 在触发器中新增强校验：存在 `END` 时必须存在 `START`。
  - 修复 `day_summaries` 约束：允许“请假日 + INCOMPLETE + adjust_minutes=0 + work_minutes=NULL”。
  - 为 `day_summaries` 与 `month_summaries` 增加 `version` 字段，补齐幂等同步口径。
  - 在 `users` 增加 `pairing_code_version`。
  - 新增 `web_read_bindings` 表，承载 Web 只读绑定会话并支持配对码重置后撤销旧会话。
- 影响：
  - 数据草案对需求中的“缺上班禁止下班、请假口径、同步幂等、配对码只读绑定”覆盖更完整，可进入 API 契约与迁移脚本阶段。

### 记录 005（数据库闭环复查与第三次修订）

- 背景：
  - 用户继续要求按需求文档确认“是否无误、是否完美闭环”。
  - 复查后确认仍存在两点风险：删除 `START` 被触发器误拦截、配对码重置后旧 Web 会话失效主要依赖应用逻辑。
- 修订动作（已完成）：
  - 将草案升级为 `Draft v0.4`。
  - 调整 `validate_punch_pair`：
    - 对软删除记录直接放行（避免阻断“删除打卡记录”场景）。
    - 仅在新增/修改 `END` 时强制要求存在 `START`。
    - 成对存在时仍强制 `END > START`。
  - 增加 `web_read_bindings` 状态一致性约束（`ACTIVE`/`REVOKED` 与 `revoked_at` 联动）。
  - 增加配对码重置撤销触发器：`trg_revoke_web_bindings_on_pairing_change`。
  - 增加配对码轮换函数：`rotate_pairing_code(...)`，统一事务内完成版本递增与会话失效。
- 影响：
  - `FR-029`（删除后重算）与 `FR-008`（缺上班禁止下班）不再冲突。
  - 配对码重置后旧 Web 只读绑定失效具备数据库层兜底，不依赖单点应用实现细节。
