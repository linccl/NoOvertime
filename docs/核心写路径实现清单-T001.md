# T001 核心写路径实现清单（P0-Batch2）

## 1. 目标与范围

本清单用于固化 P0-Batch2 新增核心写路径的统一实现基线，覆盖以下 9 个接口：

1. `POST /api/v1/migrations/requests`
2. `POST /api/v1/migrations/{migration_request_id}/confirm`
3. `POST /api/v1/migrations/forced-takeover`
4. `POST /api/v1/pairing-code/query`
5. `POST /api/v1/pairing-code/reset`
6. `POST /api/v1/recovery-code/generate`
7. `POST /api/v1/recovery-code/reset`
8. `POST /api/v1/web/read-bindings`
9. `POST /api/v1/web/read-bindings/auth`

输入依据：

- `docs/需求文档.md`（FR-021 ~ FR-028）
- `dispatch/p0_batch2_补齐核心路径.md`
- `docs/API契约草案.md`（2.2 ~ 2.10，附录 B）

统一约束：

- 统一 JSON 响应；错误响应字段为 `error_code`、`message`、`request_id`。
- 请求解析采用白名单字段；未知字段返回 `UNKNOWN_FIELD`（HTTP 400）。
- 参数格式错误/必填缺失返回 `INVALID_ARGUMENT`（HTTP 400）。
- 业务拒绝返回 HTTP 409；限流拒绝返回 HTTP 429。

## 2. 接口实现清单（可直接编码）

### 2.1 `POST /api/v1/migrations/requests`（迁移申请）

限流策略：`MIGRATION_REQUEST`

请求字段：

| 字段 | 类型 | 必填 | 约束 |
|---|---|---|---|
| `user_id` | string | 是 | UUID |
| `from_device_id` | string | 是 | UUID；必须匹配当前 writer |
| `to_device_id` | string | 是 | UUID；不得等于 `from_device_id` |
| `mode` | string | 是 | 固定 `NORMAL` |
| `expires_at` | string | 是 | RFC3339 UTC |

成功响应（HTTP 200）：

| 字段 | 类型 | 说明 |
|---|---|---|
| `migration_request_id` | string | 迁移申请 ID |
| `status` | string | `PENDING` |
| `mode` | string | `NORMAL` |
| `expires_at` | string | 过期时间（UTC） |

失败错误码闭集：

- HTTP 400: `INVALID_ARGUMENT`, `UNKNOWN_FIELD`
- HTTP 409: `MIGRATION_SOURCE_MISMATCH`, `MIGRATION_PENDING_EXISTS`, `MIGRATION_STATE_INVALID`, `MIGRATION_IMMUTABLE_FIELDS`
- HTTP 429: `RATE_LIMIT_BLOCKED`

### 2.2 `POST /api/v1/migrations/{migration_request_id}/confirm`（迁移确认）

限流策略：`MIGRATION_CONFIRM`

路径参数：

| 字段 | 类型 | 必填 | 约束 |
|---|---|---|---|
| `migration_request_id` | string | 是 | UUID |

请求字段：

| 字段 | 类型 | 必填 | 约束 |
|---|---|---|---|
| `action` | string | 是 | 固定 `CONFIRM` |
| `operator_device_id` | string | 是 | UUID；必须为旧机 writer |

成功响应（HTTP 200）：

| 字段 | 类型 | 说明 |
|---|---|---|
| `migration_request_id` | string | 迁移申请 ID |
| `status` | string | `COMPLETED` |
| `writer_device_id` | string | 新 writer 设备 |
| `writer_epoch` | integer | 切换后纪元 |
| `revoked_device_id` | string | 被撤销写入权设备 |
| `completed_at` | string | 完成时间（UTC） |

失败错误码闭集：

- HTTP 400: `INVALID_ARGUMENT`, `UNKNOWN_FIELD`
- HTTP 409: `MIGRATION_STATE_INVALID`, `MIGRATION_EXPIRED`, `MIGRATION_SOURCE_MISMATCH`, `STALE_WRITER_REJECTED`
- HTTP 429: `RATE_LIMIT_BLOCKED`

### 2.3 `POST /api/v1/migrations/forced-takeover`（强制接管）

限流策略：先 `RECOVERY_VERIFY`，再 `MIGRATION_REQUEST`

请求字段：

| 字段 | 类型 | 必填 | 约束 |
|---|---|---|---|
| `pairing_code` | string | 是 | 8 位数字 |
| `recovery_code` | string | 是 | 16 位字母数字（A-Z0-9，不区分大小写） |
| `to_device_id` | string | 是 | UUID |

成功响应（HTTP 200）：

| 字段 | 类型 | 说明 |
|---|---|---|
| `migration_request_id` | string | 迁移申请 ID |
| `status` | string | `COMPLETED` |
| `mode` | string | `FORCED` |
| `writer_device_id` | string | 新 writer 设备 |
| `writer_epoch` | integer | 切换后纪元 |
| `completed_at` | string | 完成时间（UTC） |

失败错误码闭集：

- HTTP 400: `INVALID_ARGUMENT`, `UNKNOWN_FIELD`, `PAIRING_CODE_FORMAT_INVALID`
- HTTP 409: `PAIRING_CODE_INVALID`, `RECOVERY_CODE_INVALID`, `MIGRATION_STATE_INVALID`
- HTTP 429: `RATE_LIMIT_BLOCKED`

### 2.4 `POST /api/v1/pairing-code/query`（配对码查询/首次生成）

限流策略：通用写入限流（网关层）

请求字段：

| 字段 | 类型 | 必填 | 约束 |
|---|---|---|---|
| `ensure_generated` | boolean | 是 | `true` 表示不存在则生成 |

成功响应（HTTP 200）：

| 字段 | 类型 | 说明 |
|---|---|---|
| `pairing_code` | string | 8 位数字 |
| `pairing_code_version` | integer | 版本号 |
| `pairing_code_updated_at` | string | 更新时间（UTC） |
| `is_newly_generated` | boolean | 本次是否新生成 |

失败错误码闭集：

- HTTP 400: `INVALID_ARGUMENT`, `UNKNOWN_FIELD`
- HTTP 401: `UNAUTHORIZED_DEVICE`
- HTTP 409: `STALE_WRITER_REJECTED`, `USER_NOT_FOUND`

### 2.5 `POST /api/v1/pairing-code/reset`（配对码重置）

限流策略：`PAIRING_RESET`

请求字段：

| 字段 | 类型 | 必填 | 约束 |
|---|---|---|---|
| `reason` | string | 是 | 固定枚举；MVP 至少支持 `USER_INITIATED` |

成功响应（HTTP 200）：

| 字段 | 类型 | 说明 |
|---|---|---|
| `pairing_code` | string | 新 8 位数字配对码 |
| `pairing_code_version` | integer | 新版本号 |
| `pairing_code_updated_at` | string | 更新时间（UTC） |
| `revoked_bindings_count` | integer | 被撤销的 Web 绑定数 |

失败错误码闭集：

- HTTP 400: `INVALID_ARGUMENT`, `UNKNOWN_FIELD`
- HTTP 409: `PAIRING_CODE_GENERATE_FAILED`, `USER_NOT_FOUND`, `STALE_WRITER_REJECTED`
- HTTP 429: `RATE_LIMIT_BLOCKED`

### 2.6 `POST /api/v1/recovery-code/generate`（恢复码首次生成）

限流策略：`RECOVERY_VERIFY`

请求字段：

| 字段 | 类型 | 必填 | 约束 |
|---|---|---|---|
| `require_first_time` | boolean | 是 | `true` 时仅允许首次生成 |

成功响应（HTTP 200）：

| 字段 | 类型 | 说明 |
|---|---|---|
| `recovery_code` | string | 明文恢复码（仅本次展示） |
| `recovery_code_masked` | string | 掩码展示值 |
| `shown_once` | boolean | 固定 `true` |
| `updated_at` | string | 更新时间（UTC） |

失败错误码闭集：

- HTTP 400: `INVALID_ARGUMENT`, `UNKNOWN_FIELD`
- HTTP 409: `RECOVERY_CODE_ALREADY_INITIALIZED`, `STALE_WRITER_REJECTED`
- HTTP 429: `RATE_LIMIT_BLOCKED`

### 2.7 `POST /api/v1/recovery-code/reset`（恢复码重置）

限流策略：`RECOVERY_VERIFY`

请求字段：

| 字段 | 类型 | 必填 | 约束 |
|---|---|---|---|
| `old_recovery_code` | string | 否 | 不强制重置时必填，16 位字母数字 |
| `force_reset` | boolean | 是 | `true` 表示绕过旧码校验路径 |

成功响应（HTTP 200）：

| 字段 | 类型 | 说明 |
|---|---|---|
| `recovery_code` | string | 新明文恢复码（仅本次展示） |
| `recovery_code_masked` | string | 掩码展示值 |
| `shown_once` | boolean | 固定 `true` |
| `updated_at` | string | 更新时间（UTC） |

失败错误码闭集：

- HTTP 400: `INVALID_ARGUMENT`, `UNKNOWN_FIELD`
- HTTP 409: `RECOVERY_CODE_INVALID`, `STALE_WRITER_REJECTED`
- HTTP 429: `RATE_LIMIT_BLOCKED`

### 2.8 `POST /api/v1/web/read-bindings`（Web 只读绑定创建）

限流策略：`WEB_PAIR_BIND`

请求字段：

| 字段 | 类型 | 必填 | 约束 |
|---|---|---|---|
| `pairing_code` | string | 是 | 8 位数字 |
| `client_fingerprint` | string | 是 | 客户端指纹（非空） |
| `web_device_name` | string | 是 | 设备名称（非空） |

成功响应（HTTP 200）：

| 字段 | 类型 | 说明 |
|---|---|---|
| `binding_id` | string | 绑定 ID |
| `binding_token` | string | Web 只读 token |
| `pairing_code_version` | integer | 绑定时配对码版本 |
| `status` | string | `ACTIVE` |
| `created_at` | string | 创建时间（UTC） |

失败错误码闭集：

- HTTP 400: `INVALID_ARGUMENT`, `UNKNOWN_FIELD`, `PAIRING_CODE_FORMAT_INVALID`
- HTTP 409: `PAIRING_CODE_INVALID`, `WEB_BINDING_VERSION_MISMATCH`
- HTTP 429: `RATE_LIMIT_BLOCKED`

### 2.9 `POST /api/v1/web/read-bindings/auth`（Web 只读绑定鉴权）

限流策略：`WEB_PAIR_BIND`

请求字段：

| 字段 | 类型 | 必填 | 约束 |
|---|---|---|---|
| `binding_token` | string | 是 | 非空；符合服务端 token 格式 |
| `client_fingerprint` | string | 是 | 非空；需与绑定一致 |

成功响应（HTTP 200）：

| 字段 | 类型 | 说明 |
|---|---|---|
| `binding_id` | string | 绑定 ID |
| `user_id` | string | 用户 ID |
| `status` | string | `ACTIVE` |
| `pairing_code_version` | integer | 当前版本 |
| `last_seen_at` | string | 最近访问时间（UTC） |

- HTTP 401: `UNAUTHORIZED_WEB_TOKEN`
- HTTP 409: `WEB_BINDING_REACTIVATE_DENIED`, `WEB_BINDING_VERSION_MISMATCH`
- HTTP 429: `RATE_LIMIT_BLOCKED`

## 3. FR-021 ~ FR-028 覆盖映射

| FR | 需求摘要 | 对应接口 |
|---|---|---|
| FR-021 | 生成配对码供 Web 只读绑定 | `POST /api/v1/pairing-code/query`, `POST /api/v1/web/read-bindings` |
| FR-022 | 配对码重置后旧码失效 | `POST /api/v1/pairing-code/reset`, `POST /api/v1/web/read-bindings` |
| FR-023 | 普通换机迁移（旧机确认） | `POST /api/v1/migrations/requests`, `POST /api/v1/migrations/{migration_request_id}/confirm` |
| FR-024 | 配对码默认掩码、可明文切换 | 客户端交互需求；服务端接口支撑 `pairing_code/query` 与 `pairing_code/reset` |
| FR-025 | 配对码一键复制 | 客户端交互需求；服务端提供可复制明文返回 |
| FR-026 | 恢复码生成 | `POST /api/v1/recovery-code/generate` |
| FR-027 | 恢复码仅展示一次，遗失需重置 | `POST /api/v1/recovery-code/generate`, `POST /api/v1/recovery-code/reset` |
| FR-028 | 强制迁移（配对码+恢复码） | `POST /api/v1/migrations/forced-takeover` |

## 4. 实现边界与验收口径

- 本文档仅固化“可编码接口清单与契约口径”，不替代详细数据库 DDL 与触发器实现。
- 业务错误码以“闭集”实现，禁止返回未登记的新错误码。
- 限流场景必须在接口层可观察到 `RATE_LIMIT_BLOCKED`（HTTP 429）。
- 迁移/配对码/恢复码/Web 绑定接口均需接入统一 `request_id` 错误响应。
