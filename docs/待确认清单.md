# 待确认清单

## 已按 token-only 基线收敛

- 移动端受保护接口统一为 `Authorization: Bearer <token>`。
- `POST /api/v1/tokens/issue` / `POST /api/v1/tokens/rotate` 已落地。
- `POST /api/v1/sync/commits` 顶层不再要求客户端上传 `user_id/device_id/writer_epoch`，首次成功同步会回填 `user_id`。
- 除 `tokens/*` 与 `sync/commits` 外，匿名 token 调用移动端受保护接口统一返回 `409 USER_ID_NOT_READY`。
- 迁移接管新路径优先为 `POST /api/v1/migrations/takeover`，旧路径 `POST /api/v1/migrations/forced-takeover` 保留兼容。

## 已过时口径（阅读旧文档时请忽略）

- 客户端显式上传 `X-User-ID` / `X-Device-ID` / `X-Writer-Epoch` 作为移动端写接口必需头。
- `sync/commits` 请求体顶层必须携带 `user_id/device_id/writer_epoch`。
- `migrations/requests` 必须由客户端显式上传 `user_id/from_device_id`。
- `migrations/confirm` 必须由客户端显式上传 `operator_device_id`。
- `migrations/forced-takeover` 是唯一有效接管路径，且必须由客户端显式上传 `to_device_id`。

## 后续建议

- 若前后端与文档都完成切换，可在后续迭代中删除 `forced-takeover` 兼容路径说明，仅保留 `takeover` 作为正式口径。
- 若历史草案仍保留旧示例，统一以 `docs/安卓端TokenOnly改造对API端总览.md` 与当前实现为准。
