### 2.4 空库判定（已执行）

输入（SQL）：

```sql
WITH ext_owned AS (
  SELECT objid
  FROM pg_depend
  WHERE deptype = 'e'
),
non_system_objects AS (
  SELECT 'table' AS obj_type, c.oid
  FROM pg_class c
  JOIN pg_namespace n ON n.oid = c.relnamespace
  WHERE n.nspname NOT IN ('pg_catalog', 'information_schema')
    AND c.relkind IN ('r','p','v','m','S','i')
  UNION ALL
  SELECT 'type' AS obj_type, t.oid
  FROM pg_type t
  JOIN pg_namespace n ON n.oid = t.typnamespace
  WHERE n.nspname NOT IN ('pg_catalog', 'information_schema')
    AND t.typtype = 'e'
  UNION ALL
  SELECT 'function' AS obj_type, p.oid
  FROM pg_proc p
  JOIN pg_namespace n ON n.oid = p.pronamespace
  WHERE n.nspname NOT IN ('pg_catalog', 'information_schema')
)
SELECT obj_type, count(*) AS cnt
FROM non_system_objects o
LEFT JOIN ext_owned eo ON eo.objid = o.oid
WHERE eo.objid IS NULL
GROUP BY obj_type
ORDER BY obj_type;
```

预期结果：

- 所有统计项 `cnt = 0`

实际结果：模板 SQL 统计出 `table=39`（包含 `pg_toast` 系统对象）；按“排除 `pg_%` 与 `information_schema`”修正后统计为 0。

### 4.4 `pg_catalog` 实际查询（已执行）

输入（SQL）：

```sql
-- extension
SELECT e.extname, n.nspname AS extnamespace
FROM pg_extension e
JOIN pg_namespace n ON n.oid = e.extnamespace
ORDER BY 1;

-- enum
SELECT n.nspname AS schema_name, t.typname AS enum_name
FROM pg_type t
JOIN pg_namespace n ON n.oid = t.typnamespace
WHERE t.typtype = 'e' AND n.nspname = 'public'
ORDER BY 1,2;

-- tables
SELECT table_schema, table_name
FROM information_schema.tables
WHERE table_schema = 'public' AND table_type='BASE TABLE'
ORDER BY 1,2;

-- indexes
SELECT schemaname, tablename, indexname
FROM pg_indexes
WHERE schemaname='public'
ORDER BY 1,2,3;

-- constraints
SELECT n.nspname AS schema_name, c.relname AS table_name, con.conname, con.contype
FROM pg_constraint con
JOIN pg_class c ON c.oid = con.conrelid
JOIN pg_namespace n ON n.oid = c.relnamespace
WHERE n.nspname='public'
ORDER BY 1,2,3;

-- functions
SELECT n.nspname AS schema_name,
       p.proname,
       pg_get_function_identity_arguments(p.oid) AS argument_types
FROM pg_proc p
JOIN pg_namespace n ON n.oid = p.pronamespace
WHERE n.nspname='public'
ORDER BY 1,2,3;

-- triggers
SELECT n.nspname AS schema_name, c.relname AS table_name, t.tgname
FROM pg_trigger t
JOIN pg_class c ON c.oid = t.tgrelid
JOIN pg_namespace n ON n.oid = c.relnamespace
WHERE n.nspname='public' AND NOT t.tgisinternal
ORDER BY 1,2,3;
```

实际结果（摘要）：

- `extension`：`pgcrypto(public)`、`plpgsql(pg_catalog)`。
- 核心对象命中计数：`enum=8`、`table=10`、`index=15`、`function=10`、`trigger=10`、`extension=1`。
- 核心对象 `schema`（extension 除外）均为 `public`。

### 4.5 A/B 差异结论

- 静态双基线（A vs B）对比结论：
  - extension：一致（`pgcrypto`）
  - enum/table/index/function/trigger：对象名一致
  - constraints：A 的未命名约束已全量映射到 B 的显式命名约束
  - 核心业务对象 `schema`（extension 除外）：均为 `public`
- `pg_catalog` 实库核验结论：核心对象域一致。
- 最终判定：**静态一致；实库一致（核心对象域）**
