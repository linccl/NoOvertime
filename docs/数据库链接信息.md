## 3. 可直接执行命令（按当前确认口径）

推荐使用环境变量注入密码，避免密码出现在 shell 历史中。

```bash
PGPASSWORD='<YOUR_PASSWORD>' psql \
  "host=45.207.209.114 port=5432 dbname=postgres user=user_2me8xA sslmode=prefer connect_timeout=5" \
  -c "select current_database(), current_user;"
```

如需强制 TLS，可改用：

```bash
PGPASSWORD='<YOUR_PASSWORD>' psql \
  "host=45.207.209.114 port=5432 dbname=postgres user=user_2me8xA sslmode=require connect_timeout=5" \
  -c "select current_database(), current_user;"
```

也可以使用 `~/.pgpass`（避免把密码写入命令行），示例（占位符）：

```bash
# ~/.pgpass
45.207.209.114:5432:postgres:user_2me8xA:<YOUR_PASSWORD>
chmod 0600 ~/.pgpass
```

## 4. 风险与阻塞

- 口径风险：若远端未使用 `postgres` 作为可登录库，需改为实际 `dbname`。
- TLS 风险：当前 `sslmode=prefer` 兼容性高但安全性弱于 `require/verify-full`。
- 环境阻塞：当前执行沙箱不允许外网 TCP 连接（连接 `45.207.209.114:5432` 返回 `Operation not permitted`）。
- 工具阻塞：当前环境无 `psql` 客户端，无法直接执行 PostgreSQL 登录校验。

## 5. 安全注意事项

- 不要在命令行直接拼接明文密码参数（优先 `PGPASSWORD` 或 `.pgpass`）。
- 不要将生产凭据写入公开日志、截图或版本库提交记录。
- 若后续补齐 `sslmode=require/verify-full`，需同步校验证书链与主机名匹配。
