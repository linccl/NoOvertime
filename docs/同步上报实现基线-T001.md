# T001 同步契约与业务规则基线（`POST /api/v1/sync/commits`）

## 1. 目标与范围

本基线仅用于 P0-Batch1 的同步写路径实现输入，覆盖以下内容：

- 请求/响应字段矩阵（必填、枚举、约束、未知字段策略）
- `gate_result` 与 `gate_reason` 判定矩阵
- 业务规则与错误码映射（覆盖任务清单要求的全部错误场景）

输入来源：

- `docs/需求文档.md` 第 6、8 节
- `dispatch/p0_batch1_tasks.md`
- `docs/API契约草案.md` 2.1 与附录 B/C/D/F/G
- `db/migrations/001_init.sql`

## 2. 请求字段矩阵

### 2.1 顶层字段

| 字段 | 类型 | 必填 | 约束/格式 | 备注 |
|---|---|---|---|---|
| `user_id` | string | 是 | UUID | 目标用户 |
| `device_id` | string | 是 | UUID | 提交设备 |
| `writer_epoch` | integer | 是 | `>=1` | 写入端纪元 |
| `sync_id` | string | 是 | UUID | 幂等键 |
| `payload_hash` | string | 是 | 64 位小写十六进制 SHA-256 | 规范化载荷（不含 `payload_hash` 字段本身）的 SHA-256 |
| `punch_records` | array | 是 | 可为空数组 | 打卡记录批次 |
| `leave_records` | array | 是 | 可为空数组 | 请假记录批次 |
| `day_summaries` | array | 是 | 可为空数组 | 日汇总批次 |
| `month_summaries` | array | 是 | 可为空数组 | 月汇总批次 |

### 2.2 `punch_records[]`

| 字段 | 类型 | 必填 | 枚举/约束 |
|---|---|---|---|
| `id` | string | 是 | UUID |
| `local_date` | string | 是 | `YYYY-MM-DD` |
| `type` | string | 是 | `START` / `END` |
| `at_utc` | string | 是 | UTC 时间；分钟粒度（秒/毫秒必须为 0） |
| `timezone_id` | string | 是 | IANA 时区名 |
| `minute_of_day` | integer | 是 | `0..1439` |
| `source` | string | 是 | `AUTO` / `MANUAL` / `MAKEUP` / `EDIT` |
| `deleted_at` | string/null | 是 | `null` 或 UTC 时间 |
| `version` | integer | 是 | `>=1` |

### 2.3 `leave_records[]`

| 字段 | 类型 | 必填 | 枚举/约束 |
|---|---|---|---|
| `id` | string | 是 | UUID |
| `local_date` | string | 是 | `YYYY-MM-DD` |
| `leave_type` | string | 是 | `AM` / `PM` / `FULL_DAY` |
| `deleted_at` | string/null | 是 | `null` 或 UTC 时间 |
| `version` | integer | 是 | `>=1` |

### 2.4 `day_summaries[]`

| 字段 | 类型 | 必填 | 枚举/约束 |
|---|---|---|---|
| `id` | string | 是 | UUID |
| `local_date` | string | 是 | `YYYY-MM-DD` |
| `start_at_utc` | string/null | 否 | UTC 时间 |
| `end_at_utc` | string/null | 否 | UTC 时间 |
| `is_leave_day` | boolean | 是 | - |
| `leave_type` | string/null | 否 | `AM` / `PM` / `FULL_DAY` |
| `is_late` | boolean/null | 否 | - |
| `work_minutes` | integer/null | 否 | `>=0` |
| `adjust_minutes` | integer/null | 否 | 30 分钟步进；请假日固定 0 |
| `status` | string | 是 | `INCOMPLETE` / `COMPUTED` |
| `version` | integer | 是 | `>=1` |
| `updated_at` | string | 是 | UTC 时间 |

### 2.5 `month_summaries[]`

| 字段 | 类型 | 必填 | 枚举/约束 |
|---|---|---|---|
| `id` | string | 是 | UUID |
| `month_start` | string | 是 | 月首日期（`YYYY-MM-01`） |
| `work_minutes_total` | integer | 是 | `>=0` |
| `adjust_minutes_balance` | integer | 是 | 可正可负 |
| `version` | integer | 是 | `>=1` |
| `updated_at` | string | 是 | UTC 时间 |

### 2.6 未知字段策略（强约束）

- 顶层对象及所有子对象均采用白名单字段。
- 出现任意未知字段，立即返回 `UNKNOWN_FIELD`（HTTP 400）。
- 禁止“透传后忽略未知字段”。

## 3. 响应字段矩阵

### 3.1 写入生效/NOOP

| 字段 | 类型 | 必填 | 规则 |
|---|---|---|---|
| `request_id` | string | 是 | 回显请求头 `X-Request-ID`；未传入时服务端生成（32 位小写 hex） |
| `gate_result` | string | 是 | `APPLIED` / `NOOP` |
| `gate_reason` | string | 是 | 见第 4 节 |
| `sync_commit.sync_id` | string | 是 | 与请求一致 |
| `sync_commit.status` | string | 是 | `APPLIED`（NOOP 复用既有记录时保持既有值） |
| `sync_commit.created_at` | string | 是 | UTC 时间 |

### 3.2 拒绝/失败

| 字段 | 类型 | 必填 | 规则 |
|---|---|---|---|
| `error_code` | string | 是 | 见第 5 节 |
| `message` | string | 是 | 面向调用方的错误描述 |
| `request_id` | string | 是 | 回显请求头 `X-Request-ID`；未传入时服务端生成（32 位小写 hex） |
| `gate_result` | string | 条件必填 | 仅门控冲突场景返回，固定 `REJECTED` |
| `gate_reason` | string | 条件必填 | 仅门控冲突场景返回，固定 `SYNC_ID_CONFLICT` |

## 4. 门控判定矩阵（固定）

| 判定条件 | HTTP | gate_result | gate_reason | 外部错误码 |
|---|---|---|---|---|
| 相同 `sync_id` + 相同 `payload_hash` | 200 | `NOOP` | `REPLAY_NOOP` | 无 |
| 相同 `sync_id` + 不同 `payload_hash` | 409 | `REJECTED` | `SYNC_ID_CONFLICT` | `SYNC_ID_CONFLICT` |
| 新 `sync_id` + 低/同版本 | 200 | `NOOP` | `LOW_OR_EQUAL_VERSION` | 无 |
| 写入生效（至少 1 项变更） | 200 | `APPLIED` | `APPLIED_WRITE` | 无 |

## 5. 业务规则与错误码映射（T001 范围）

| 场景 | 触发条件 | 来源规则/约束 | API 返回 |
|---|---|---|---|
| 参数非法 | 类型错误、必填缺失、格式不合法（如 UUID/日期/枚举） | 请求层校验 | `INVALID_ARGUMENT` |
| 未知字段 | 白名单外字段出现 | 请求层白名单策略 | `UNKNOWN_FIELD` |
| `sync_id` 冲突 | 相同 `sync_id` + 不同 `payload_hash` | `RULE_SYNC_ID_CONFLICT` / `uq_sync_commits_user_sync` | `SYNC_ID_CONFLICT` + `gate_result=REJECTED` + `gate_reason=SYNC_ID_CONFLICT` |
| 低/同版本 NOOP | 新 `sync_id` 但实体 `version` 低于或等于现存版本 | `RULE_SYNC_LOW_OR_EQUAL_VERSION` | HTTP 200 + `gate_result=NOOP` + `gate_reason=LOW_OR_EQUAL_VERSION`（不返回 `error_code`） |
| 缺 START 禁 END | 当日存在 END 但无 START | `P0001 + PUNCH_END_REQUIRES_START` | `PUNCH_END_REQUIRES_START` |
| END 不晚于 START | 同日 `END <= START` | `P0001 + PUNCH_END_NOT_AFTER_START` | `PUNCH_END_NOT_AFTER_START` |
| 全天请假禁自动打卡 | `FULL_DAY` 与 `AUTO` 打卡冲突（双向） | `P0001 + AUTO_PUNCH_ON_FULL_DAY_LEAVE` 或 `P0001 + FULL_DAY_LEAVE_WITH_AUTO_PUNCH` | `CONFLICT_AUTO_PUNCH_FULL_DAY_LEAVE` |
| 时间精度非法 | `at_utc` 非分钟粒度（秒/毫秒非 0） | 精度校验 / `ck_punch_records_at_utc_minute_precision` | `TIME_PRECISION_INVALID` |
| 时间字段不一致 | `local_date` 与 `timezone_id/at_utc` 不一致，或 `minute_of_day` 与 `at_utc` 不一致 | `ck_punch_records_local_date_match_timezone` / `ck_punch_records_minute_of_day_match_at_utc` | `TIME_FIELDS_MISMATCH` |
| 非当前写入端提交 | `device_id != users.writer_device_id` 或 `writer_epoch != users.writer_epoch` | `P0001 + SYNC_COMMIT_STALE_WRITER` | `STALE_WRITER_REJECTED` |

## 6. 原子提交边界（实现约束）

同一事务内必须共同提交：

- `punch_records`
- `leave_records`
- `day_summaries`
- `month_summaries`
- `sync_commits`

任一环节失败，整体回滚，不允许部分生效。
