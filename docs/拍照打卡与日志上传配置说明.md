# 拍照打卡与日志上传配置说明

## 新增接口
- `POST /api/v1/punch-photos/upload`
  - Header: `Authorization: Bearer <mobile_token>`
  - Form:
    - `punch_record_id`
    - `local_date`
    - `punch_type`
    - `file`
- `POST /api/v1/logs/upload`
  - Header: `Authorization: Bearer <mobile_token>`
  - Form:
    - `log_date`
    - `file`

成功响应统一返回：

```json
{
  "url": "https://...",
  "object_key": "punch-photos/... or logs/...",
  "expires_at": "2026-05-14T10:00:00Z",
  "request_id": "..."
}
```

## 存储后端
- 默认 `local`
- 可切换为 `oss`

## 配置项
- `UPLOAD_STORAGE_BACKEND`
  - `local` 或 `oss`
- `UPLOAD_LOCAL_DIR`
  - 本地存储目录；`local` 模式必填，默认 `./uploads-data`
- `UPLOAD_PUBLIC_BASE_URL`
  - 公开访问前缀；本地模式会拼成 `${base}/uploads/...`
- `UPLOAD_OSS_ENDPOINT`
- `UPLOAD_OSS_BUCKET`
- `UPLOAD_OSS_ACCESS_KEY_ID`
- `UPLOAD_OSS_ACCESS_KEY_SECRET`
- `UPLOAD_OSS_PREFIX`

## 保留策略
- 服务端为照片与日志记录 `expires_at`
- 服务端启动后会跑后台清理循环，同时每次成功上传后也会顺带清理已过期对象
- 已过期对象删除后，对应数据库记录会标记 `deleted_at`
