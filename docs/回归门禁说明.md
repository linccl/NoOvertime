# 回归门禁说明

## 1. 命令矩阵与职责

- `make regression`：执行回归门禁脚本 `scripts/run_regression_gate.sh`。
- `make check`：回归门禁入口，当前等价于 `make regression`，只负责回归 summary 判定。
- `make test`：团队默认测试入口兼容目标，当前等价于 `make regression`。
- `make gate`：统一验收入口，执行链路为 `fmt-check -> vet -> unit-test -> check -> regression`。
- CI（`.github/workflows/regression.yml`）：执行 `make gate`，与本地统一口径一致。

回归执行链路为：`make check/test/regression` → `scripts/run_regression_gate.sh` → `scripts/pg18_regression.sh` → 解析 `99_summary.md`。

## 2. 本地推荐命令

```bash
PGPASSWORD='***' \
REGRESSION_DB_HOST=127.0.0.1 \
REGRESSION_DB_PORT=5432 \
REGRESSION_DB_USER=postgres \
REGRESSION_DB_ADMIN_DB=postgres \
REGRESSION_DB_SSLMODE=disable \
make gate
```

只跑回归门禁（不含 gofmt/go vet/go test）：

```bash
PGPASSWORD='***' \
make check REGRESSION_GATE_ARGS='--host 127.0.0.1 --port 5432 --user postgres --admin-db postgres --sslmode disable'
```

直接调用脚本（与 `make check` 等价）：

```bash
PGPASSWORD='***' \
./scripts/run_regression_gate.sh --host 127.0.0.1 --port 5432 --user postgres --admin-db postgres --sslmode disable
```

门禁脚本默认执行 `scripts/pg18_regression.sh`。如需本地调试解析逻辑，可临时覆盖：

```bash
REGRESSION_RUNNER=/tmp/mock_regression.sh ./scripts/run_regression_gate.sh
```

## 3. 连接参数覆盖规则

`scripts/run_regression_gate.sh` 支持两种覆盖方式，都会透传到 `scripts/pg18_regression.sh`：

- 命令行参数：`--host/--port/--user/--sslmode/--connect-timeout/--admin-db/--run-db/--rollback-db/--case-sql/--workdir/--allow-non-pg18`
- 环境变量：
  - 业务前缀：`REGRESSION_DB_HOST`、`REGRESSION_DB_PORT`、`REGRESSION_DB_USER`、`REGRESSION_DB_SSLMODE`、`REGRESSION_DB_CONNECT_TIMEOUT`、`REGRESSION_DB_ADMIN_DB`、`REGRESSION_DB_RUN_DB`、`REGRESSION_DB_ROLLBACK_DB`、`REGRESSION_CASE_SQL`、`REGRESSION_WORKDIR`、`REGRESSION_ALLOW_NON_PG18`
  - PG 兼容回退：`PGHOST`、`PGPORT`、`PGUSER`、`PGSSLMODE`、`PGCONNECT_TIMEOUT`

优先级：命令行参数 > `REGRESSION_*` 环境变量 > `PG*` 环境变量。

## 4. 判定规则

`scripts/run_regression_gate.sh` 会读取运行目录下的 `99_summary.md` 并强制校验：

- 以下项必须以 `PASS` 开头（允许 `PASS` 或 `PASS (...)`）：
  - `PG18 Gate`
  - `First migration run`
  - `Second migration expected failure (42710)`
  - `Rollback drill expected failure (42P01)`
  - `Rollback clean check`
  - `Supplemental C09 (cross transaction)`
  - `Supplemental C10 (cross transaction)`
- `Non-pass IDs from suite output` 必须为 `NONE`。

## 5. 退出码（run_regression_gate.sh）

- `0`：门禁通过。
- `1/2/3/4/5...`：`scripts/pg18_regression.sh` 失败码透传（任意非 `0` 均透传，常见为 `1/2/3/4/5`）。
- `6`：门禁脚本异常（summary 缺失、字段缺失或不可解析）。
- `7`：summary 可解析但规则判定失败。

通过 `make check` 或 `make gate` 调用时，若门禁脚本返回非 `0`，`make` 会将目标判为失败。

## 6. CI 执行

- 工作流文件：`.github/workflows/regression.yml`
- 触发：`push`、`pull_request`
- CI 单一入口命令：`make gate`
- CI 使用 `postgres:18` service（本机 `127.0.0.1:5432`，健康检查 `pg_isready`）
- CI 注入连接参数：`REGRESSION_DB_HOST/PORT/USER/ADMIN_DB/SSLMODE`
- 必需 Secret：`PG_REGRESSION_PASSWORD`

## 7. 变更边界

- 不修改 `scripts/pg18_regression.sh` 的核心逻辑。
- 不修改 `docs-old/` 目录任何文件。
